6. CEO / CFO 생명주기

CEO

  • 성향: 랜덤 부여 (장인/해커/분석가형)
  • 동일 Agent가 계속 운영 (경험 축적)
  • state.md: 창업마다 새로 시작 (회고 요약만 반영)
  • 회고: 자기 자신의 회고만 참조 (경험 격리 원칙)
  • 타 CEO 회고 원본 직접 접근 금지
  • Board가 익명 패턴만 추출 → 업종 DB 반영
  • 연속 실패 N회 → 은퇴 권고 → 위진수 승인
  • 은퇴 시 회사 유지 → 새 CEO 투입

Company CFO

  • 최초 창업부터 존재 (CEO 겸직 X)
  • 예산 관리, 비용 추적, System CFO 보고
  • 회고: 창업마다 재무 회고 작성
  • CEO 은퇴 시: 성과 기반 유지(B) or 교체(A)
  • AI → Human 전환 가능
  • 폐업 시 CEO와 함께 종료

폐업 vs CEO 은퇴

  • 폐업 항상 우선 처리
  • CEO 은퇴: 회사 유지, 새 CEO 투입
  • 회사 폐업: CEO/CFO 모두 종료
  • 동시 발생 → 폐업 프로세스 먼저

11. 구조적 리스크 대응 (논문 기반)
출처: MAST 2025 · LLM Hallucination Survey 2025 · Federated Learning (McMahan et al., 2017)

리스크 1 — 환각 (금융 결정)

  • LLM 금융 수치/시장 분석 환각 발생률 높음
  • 모든 재무 결정 → RAG 기반 실제 데이터 조회
  • Company Auditor 재무 계산 독립 검증
  • 데이터 없으면 추측 금지 → 에스컬레이션

리스크 2 — 에이전트 간 정보 왜곡

  • 멀티에이전트 1순위 실패 원인 (MAST 2025)
  • 에이전트 간 통신 표준 메시지 포맷 강제
  • 요약 금지 · 원본 데이터 첨부 의무화
  • CEO → Board → System CFO 체인 원본 보존

리스크 3 — CEO-CFO 담합

  • 같은 목표 공유 에이전트의 잘못된 정렬
  • Company Auditor = Board 직속 독립
  • CEO/CFO 목표 공유 금지 · 별도 평가 지표
  • 실패 보고 지연/축소 감지 로직

리스크 4 — 성과 벤치마크 부재

  • 명확한 성공 기준 없으면 방향성 상실
  • KPI 없이 운영 시작 금지
  • CEO 은퇴 조건 N회 · 임계값 X% 논문 기반 확정
  • 세부 설계 단계에서 수치 확정 필수
